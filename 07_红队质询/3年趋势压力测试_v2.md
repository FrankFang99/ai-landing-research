## 1. v2 评估框架（5 维压力测试）

每个赛道都要过这 5 道坎（v1 是 4 维，v2 新增"开源冲击"维度）：

| 维度 | 问题 | 红线（PASS 难） |
|---|---|---|
| **技术拐点** | 3 年内会不会被新模型 / 新范式一夜淘汰？ | 每 6 个月一次大版本迭代 = 高风险 |
| **政策风向** | 3 年内会不会被监管叫停 / 限制？ | 涉未成年人 / 金融 / 医疗 = 高风险 |
| **商业可持续** | 3 年内能不能算过 ROI？ | 算不过账 / 客户不愿付费 = 高风险 |
| **护城河** | 3 年内会不会被巨头碾压成白菜价？ | 同质化严重 / 缺差异化 = 高风险 |
| **🆕 开源冲击** | 3 年内会不会被开源模型直接打穿？ | DeepSeek R1 / Llama / Qwen 已能对标 GPT-4 = 闭源商业模式被颠覆 |

**判定标准**（v2 升级）：
- 0 个红灯 = 真趋势，**值得压**
- 1 个红灯 = 谨慎乐观，可以试点但不全压
- 2 个红灯 = 高风险，**观望不押注**
- 3+ 红灯 = 炒作，**现在就该撤**

**v2 红队新发现**：**"开源冲击"是 2025-2026 年最容易被忽略的风险维度**。DeepSeek R1 用 29.4 万美元训练成本实现 GPT-4 性能，**英伟达单日蒸发 6000 亿美元市值**（2025-01）——**这就是"开源冲击"对闭源商业模式的直接颠覆**。**任何"靠闭源 API 收差价"的赛道，3 年内都会被开源打穿**。

### 场景 1：AI 客服 / 智能问答

| 维度 | 评级 | v2 说明（vs v1 升级） |

| 技术拐点 | 🟡 黄 | LLM 推理成本持续下降 + RAG 标准化，技术成熟；但每 6 个月新模型一迭代，老方案快速贬值 |

| 护城河 | 🔴 红（v1 已是红） | 巨头（百度 / 阿里 / 字节）已有成熟产品，咨询公司很难差异化；按效果付费模式（Sierra / 蚂蚁数科）挤压定制空间 |
| 🆕 开源冲击 | 🔴 红（v2 新增） | DeepSeek / Qwen / Llama 已能对标 GPT-4，**通用客服 API 已被开源打穿**——咨询公司只剩"行业垂类客服"窄定位 |

**总评**：🔴 **2 红 2 黄 1 绿**（v1 是 1 红 2 黄 1 绿）——**升级为高风险**。

**v2 红队注（与 v1 不同）**：
- v1 我判定"谨慎乐观"，建议"只做行业垂类"；
- v2 **升级为高风险**——**MIT 报告 + Gartner 双数据 + Klarna 2024 回退**三个独立信源共同印证；
- 销售打单要更谨慎：**只接"行业垂类 + copilot 模式"**，**不接"通用客服 + autopilot"**。

**销售建议**：**只接金融 / 政务 / 医疗等垂类客服**，**不接通用客服**。每个项目都要有"具体行业知识图谱"——这是开源冲不穿的护城河。
